Jupiter — Gamification PRD

Behavioural product design to turn budgeting into a weekly ritual — verbatim from the PRD. :contentReference[oaicite:1]{index=1}

1. Strategic Context

Jupiter’s long-term strategic moat cannot be built on payments, which are fully commoditised, nor on banking rails, which are standardised. The real moat lies in habit formation: creating a financial behaviour ecosystem where users repeatedly return, not because they must, but because the system makes managing money feel rewarding, clarifying, and emotionally reinforcing.

Budgeting should be that anchor habit, but current user behaviour demonstrates the opposite pattern: strong initial intent → rapid decay → long-term disengagement.

The strategic imperative:

  • If Jupiter can turn budgeting into a weekly ritual, it unlocks a multi-year LTV flywheel.
  • Build a behaviour-shaping financial layer that translates raw spending into emotionally resonant insights, action-ready triggers, and weekly progress loops.

2. Problem Statement - The Behavioural Gap

The problem is not that users “don’t budget.” The problem is that Jupiter’s current structure asks users to do the hard part of budgeting—interpretation and discipline—without providing the emotional scaffolding required to sustain it.

There are four intertwined behavioural gaps:

2.1 Cognitive Interpretation Gap

Users do not struggle with numbers; they struggle with meaning. A category total of ₹3,450 is inert information. A contextual insight like “up 44% vs last week” is a narrative.

2.2 Motivation Gap

Discipline has no immediate reward; indulgence does. Unless the system creates micro-reward loops, intention always loses to temptation.

2.3 Action Gap

Even when meaning is clear, action is not. “Your food spending increased” is merely information. “Set a ₹2,500 weekly food budget?” is an actionable trigger.

2.4 Feedback Loop Gap

Humans build habits through tight feedback loops. Jupiter’s budgeting currently provides no weekly closure, no emotional reinforcement, no visible progress and no comparative context. The user is left floating in ambiguity.

3. Product Vision

Build a behaviour-shaping financial layer that translates raw spending into emotionally resonant insights, action-ready triggers, and weekly progress loops—enabling young Indians to internalize financial discipline as part of their identity.

This is not “gamification” in the entertainment sense. This is behavioural infrastructure, similar to what Duolingo did for learning or Strava did for fitness.

4. Proposed Solution - The Behaviour Architecture

The solution consists of four interlocking systems, each reinforcing the others. Each is described deeply but clearly, ensuring the narrative remains digestible.

4.1 Insight Reframing Layer

Replace passive category totals with interpretive insights: week-over-week changes, percentage deltas, contextual interpretation, spend anomalies, and behaviour-based micro-narratives.

4.2 Weekly Budget Challenge Engine

Offer a pre-framed prompt like: “Set a Food challenge for next week? Recommended budget: ₹2,500.” User makes a binary decision: yes or no. Once activated, this becomes a weekly loop with real-time tracking, nudges at 70% and 100%, win/loss evaluation on Sunday, and an automatic reset prompt.

4.3 Social Benchmarking Layer

Use soft benchmarking (percentile bands, city-level comparison, “Among users with similar budgets” framing) to provide social proof without shaming.

4.4 Emotional Feedback Layer

Use confetti micro-interactions, haptic taps, expressive emojis, warm copy and weekly progress cards to create affective memory—turning budgeting into a warm ritual.

5. Technical Architecture

Designed to be modular, scalable, and low-latency. Key services: Insights Service, Budget Engine, Nudge Engine, Ranking Engine, Report Card Service. Data requirements: ≥90% categorisation accuracy, transaction streaming, weekly aggregation, real-time nudge eligibility.

6. Measurement Framework

Measure a behavioural funnel: Awareness (view insights weekly), Activation (set challenges), Engagement (return midweek), Behavioural Outcome (reduced overspend), Retention (complete 4+ weekly cycles). Measure causally—does viewing insights increase challenge adoption, etc.

7. Risks & Tradeoffs

Risks: over-nudging (fatigue), benchmarking (shame), categorisation errors (distrust), emotional overload (infantilisation). Mitigations: nudge caps, positive framing, manual recategorisation flows, mature tone and restrained animation.

8. Rollout Plan

Dogfood internally → Bangalore Android (40%) → Mumbai Android (40%) → iOS → Pan-India. V2: streaks, difficulty adjustments, cross-category challenges.
